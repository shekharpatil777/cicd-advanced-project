## 3. Windows Compatibility Fixes

### Step 3.1: Fixed Symlink Privilege Error
- Original code used os.symlink() for latest model reference
- Windows requires administrator privileges for symlinks
- Solution: Replaced with JSON-based latest.json file containing model path
- Updated train.py to use version_dir.as_posix() for cross-platform paths

### Step 3.2: Path Handling
- Ensured all file paths use forward slashes for Linux container compatibility
- Used pathlib.Path for cross-platform path operations

## 7. Key Technical Decisions

### Decision 7.1: JSON-based Latest Reference
- Chose JSON over symlinks for Windows compatibility
- Ensures cross-platform compatibility (Windows development, Linux deployment)
- Maintains versioning information and metadata

### Decision 7.2: Integrated vs Separate Services
- Initially created separate mlops/predict.py service
- Later integrated ML endpoints into main app/app.py
- Simplifies deployment and reduces container complexity

### Decision 7.3: Python Version Upgrade
- Upgraded from Python 3.9 to 3.11 in Docker
- Required for scikit-learn 1.8.0 compatibility
- Ensures access to latest ML library features

## 14. Commands Used

### Training and Testing
```bash
# Install ML dependencies
pip install -r mlops/requirements.txt

# Train ML model
python mlops/train.py

# Test model loading
python -c "from mlops.predict import load_model; load_model()"

# Test Flask app
python -c "from app.app import app; print('App loads successfully')"
```

### Docker Operations
```bash
# Build Docker image
docker build -t cicd-app:latest .

# Run Docker container
docker run --rm -p 5000:5000 cicd-app:latest

# Run full stack
docker-compose up -d
```

### API Testing
```bash
# Health check
curl http://localhost:5000/health

# Model info
curl http://localhost:5000/model-info

# Single prediction
curl -X POST http://localhost:5000/predict \
  -H "Content-Type: application/json" \
  -d '{"age": 35, "income": 50000, "credit_score": 700, "loan_amount": 200000, "employment_years": 10}'
```

## 15. Troubleshooting Notes
- If model loading fails: Check latest.json path format (should use forward slashes)
- If Docker build fails: Ensure Python version compatibility with ML packages
- If predictions fail: Verify input data matches expected features
- Windows symlink errors: Use JSON-based latest reference instead
